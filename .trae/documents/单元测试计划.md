# 每次代码修改后执行单元测试计划

## 目标
建立规范：每次代码修改后，必须运行单元测试确保功能正常。

## 当前测试文件
- `backend/test_app.py` - 后端 API 测试（11个测试用例）
- `frontend` - 前端使用 Vite 热更新，无需额外测试

## 修改后测试流程

### 1. Backend 修改后
```bash
cd c:\work\backtrader
.\venv\Scripts\pytest backend/test_app.py -v
```

### 2. Frontend 修改后
- Vite 会自动热更新
- 手动刷新浏览器验证

### 3. 修改分类与测试要求

| 修改类型 | 必须运行的测试 | 说明 |
|---------|--------------|------|
| REST API 修改 | `test_get_strategies`, `test_submit_backtest`, `test_get_backtest_result` | 验证 API 正常 |
| WebSocket 修改 | `test_websocket_connect`, `test_websocket_progress_events` | 验证 WebSocket 连接和事件 |
| 回测引擎修改 | 运行完整回测并观察日志 | 端到端测试 |
| 文件上传修改 | `test_upload_*` | 验证上传功能 |
| 图表相关修改 | 手动浏览器测试 | 需要可视化验证 |

## 实现步骤

1. **每次 Backend 修改后**：
   - 运行 `pytest backend/test_app.py -v`
   - 确保所有测试通过
   - 如果 WebSocket 相关修改，额外运行 `test_realtime.py` 验证实时功能

2. **回测引擎修改后**：
   - 运行单元测试
   - 手动运行前端回测，观察日志确认实时推送正常

3. **测试失败处理**：
   - 分析失败原因
   - 修复代码
   - 重新运行测试
   - 确认通过后再继续
