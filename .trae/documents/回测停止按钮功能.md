# 回测控制按钮功能

## 需求
回测控制需要3个按钮：
1. **开始** - 启动回测（已有）
2. **暂停** - 暂停回测（已有）
3. **停止** - 停止回测（需要新增）

## 现有实现
- 开始按钮：通过 `BacktestForm` 的 `onSubmit` 触发
- 暂停按钮：通过 `handlePause` 发送 `pause_backtest` 事件

## 需要添加的功能

### 1. 前端 - 添加停止按钮
- 修改 `frontend/src/components/BacktestForm.jsx`
- 添加 `Button` 组件用于停止回测
- 按钮样式：红色，危险操作

### 2. 前端 - 状态管理
- `frontend/src/App.jsx` 添加 `stopping` 状态
- 添加 `handleStop` 函数发送 `stop_backtest` 事件

### 3. 前端 - 发送停止事件
- 当点击停止按钮时，发送 `stop_backtest` 事件
- 事件格式：`{ task_id, client_id }`

### 4. 后端 - 接收停止事件
- `backend/app.py` 添加 `socketio.on('stop_backtest')` 处理器
- 停止回测引擎的运行

### 5. 后端 - 实现停止逻辑
- `backtest_engine.py` 添加 `stop()` 方法
- 停止 `cerebro.run()` 的执行
- 可以通过设置停止标志位或抛出异常来实现

## 修改文件
- `frontend/src/App.jsx` - 添加 stopping 状态和 handleStop
- `frontend/src/components/BacktestForm.jsx` - 添加停止按钮
- `backend/app.py` - 添加 stop_backtest 事件处理器
- `backtest_engine.py` - 添加 stop() 方法
