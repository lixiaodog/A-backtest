# 行情数据清洗模块计划

## 目标
使用 SQLite 存储行情数据，支持日线和1分钟数据。

## 存储方案

| 数据类型 | 存储方式 | 说明 |
|---------|---------|------|
| 日线 | 单表 `stock_daily` | 所有股票，单表+索引 |
| 1分钟 | 分表 `min1_{code}` | 按股票代码分表 |

## 数据库结构

### stock_daily 表
```sql
CREATE TABLE stock_daily (
    code TEXT,
    datetime TEXT,
    open REAL,
    high REAL,
    low REAL,
    close REAL,
    volume REAL,
    PRIMARY KEY (code, datetime)
);
CREATE INDEX idx_daily ON stock_daily(code, datetime);
```

### min1_{code} 表（如 min1_600519）
```sql
CREATE TABLE min1_600519 (
    datetime TEXT PRIMARY KEY,
    open REAL,
    high REAL,
    low REAL,
    close REAL,
    volume REAL
);
```

## 实现步骤

### 1. `data_fetcher.py` 模块
- `init_db()` - 初始化数据库，创建日线表
- `fetch_daily()` - 获取日线数据并存储
- `fetch_1min()` - 获取1分钟数据并存储
- `load_daily()` - 加载日线数据
- `load_1min()` - 加载1分钟数据
- `resample_1min_to_period()` - 将1分钟合成为指定周期

### 2. 数据清洗
- 列名统一：datetime, open, high, low, close, volume
- 时区处理：Asia/Shanghai
- 去重（PRIMARY KEY 自动处理）
- 数据类型转换

### 3. 合成规则
```python
df_5min = df_1min.resample('5min').agg({
    'open': 'first', 'high': 'max', 'low': 'min', 'close': 'last', 'volume': 'sum'
})
```

### 4. 命令行工具
```bash
python fetch_data.py 600519 daily 20230101 20231231  # 日线
python fetch_data.py 600519 1min 20260301 20260328    # 1分钟
```
