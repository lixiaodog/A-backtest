# 策略自注册系统设计

## 需求
后端通过扫描策略文件夹，自动发现并注册策略（分析源文件提取参数），前端根据不同策略展示不同的参数交互界面。

## 方案概述
- 策略文件存放在 `backend/strategies/` 目录
- 启动时自动扫描所有 `.py` 文件
- 通过解析文件内容（AST 或正则）提取策略元信息和参数
- 前端通过 API 获取策略列表，动态渲染参数表单

## 实现方案

### 1. 策略文件规范
策略文件放在 `backend/strategies/` 目录，文件名即为策略名。

策略文件格式示例：
```python
# sma_cross.py
class SMACrossStrategy:
    params = (
        ('fast_period', 10),  # 快速均线周期
        ('slow_period', 30),   # 慢速均线周期
    )
    
    name = '双均线交叉策略'  # 可选，不存在时使用文件名
    description = '当快速均线上穿慢速均线时买入，下穿时卖出'  # 可选，不存在时为空
```

### 2. 策略扫描与解析
- 扫描 `backend/strategies/` 目录下所有 `.py` 文件
- 解析每个文件的 AST 或正则匹配提取：
  - `name`: 策略中文名（从类属性或注释获取）
  - `description`: 策略描述
  - `params`: 参数列表（名字、默认值、注释）

### 3. 后端 API
- `GET /api/strategies` - 返回所有策略列表及参数定义

返回格式：
```json
{
  "strategies": [
    {
      "id": "sma_cross",
      "name": "双均线交叉策略",
      "description": "当快速均线上穿慢速均线时买入...",
      "params": [
        {"name": "fast_period", "label": "快速均线周期", "type": "int", "default": 10, "min": 1, "max": 100},
        {"name": "slow_period", "label": "慢速均线周期", "type": "int", "default": 30, "min": 1, "max": 200}
      ]
    }
  ]
}
```

### 4. 前端实现
- 启动时调用 `GET /api/strategies` 获取策略列表
- 根据选中的策略动态渲染对应的参数输入组件

## 修改文件
