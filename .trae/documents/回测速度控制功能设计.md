# 回测速度控制功能设计

## 需求
客户端有一个滑块，可以通过这个滑块控制回测速度：
- 最大时：不限速
- 最小时：每秒 1 个 bar
- 滑块位置：K线图表标题右边，TRADEVIEW 正上方
- 速度只影响自己这个 SESSION，不影响其它用户

## 实现步骤

### 1. 前端 - 在 TradeView 组件上方添加速度控制
- 修改 `App.jsx` 中 TradeView 上方的标题区域
- 添加 `Slider` 组件（范围 1-100，1=最慢每秒1个bar，100=最快不限速）
- 滑块显示当前速度值

### 2. 前端 - 状态管理
- 添加 `speed` state：`const [speed, setSpeed] = useState(100)`
- 当滑块值改变时，发送 `set_speed` 事件到后端

### 3. 前端 - 发送速度控制事件
- 当滑块值改变时，发送 `set_speed` 事件到后端
- 事件格式：`{ speed: number, client_id: string }`

### 4. 后端 - 接收速度控制事件
- 在 `app.py` 中添加 `socketio.on('set_speed')` 处理器
- 将速度值存储在 `backtest_tasks[client_id]` 中
- 每个客户端有独立的速度设置，互不影响

### 5. 后端 - 修改回测引擎
- 在 `RealtimeChartObserver` 中添加速度控制
- 每次 `next()` 调用时检查速度设置
- 如果速度 < 100，则添加延迟
- 公式：`delay = (100 - speed) * 0.01` 秒（speed=1 时延迟约1秒，speed=100 时延迟0）
- 使用线程锁确保线程安全

### 6. 修改文件
- `frontend/src/App.jsx` - 添加速度滑块和发送速度事件
- `backend/app.py` - 接收速度事件并存储到任务中
- `backtest_engine.py` - RealtimeChartObserver 中实现延迟逻辑

## 关键技术点
- 滑块使用 antd 的 `Slider` 组件
- 速度存储在每个任务的 `engine` 对象中
- 延迟使用 `time.sleep()` 在 observer 的 next() 中实现
- 线程安全使用 `threading.Lock`
