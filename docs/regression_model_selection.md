# 回归训练模型选择指南

## 📊 系统支持的回归模型

当前系统支持以下回归模型：

### 1. LightGBM ⭐⭐⭐⭐⭐（强烈推荐）

**优点**：
- ✅ 训练速度快
- ✅ 预测速度快
- ✅ 准确率高
- ✅ 内存占用小
- ✅ 支持GPU加速
- ✅ 能处理大规模数据
- ✅ 对回归任务优化好

**缺点**：
- ⚠️ 容易过拟合（需要调参）
- ⚠️ 对特征尺度敏感（需要归一化）
- ⚠️ 参数较多

**适用场景**：
- 预测连续值（收益率）
- 数据量大
- 需要快速训练和预测
- 追求高准确率

**推荐指数**: ⭐⭐⭐⭐⭐

---

### 2. RandomForest ⭐⭐⭐⭐

**优点**：
- ✅ 鲁棒性强，不易过拟合
- ✅ 对特征尺度不敏感
- ✅ 能处理高维数据
- ✅ 稳定性好
- ✅ 可解释性强

**缺点**：
- ⚠️ 对回归任务的优化不如LightGBM
- ⚠️ 模型较大
- ⚠️ 预测速度较慢

**适用场景**：
- 不确定最佳模型
- 需要稳定的结果
- 不想花时间调参

**推荐指数**: ⭐⭐⭐⭐

---

### 3. Ridge ⭐⭐⭐

**优点**：
- ✅ 训练速度极快
- ✅ 模型简单
- ✅ 不易过拟合
- ✅ 内存占用极小
- ✅ 可解释性强

**缺点**：
- ⚠️ 只能处理线性关系
- ⚠️ 准确率较低
- ⚠️ 对非线性数据效果差

**适用场景**：
- 线性关系明显
- 需要快速baseline
- 作为集成模型的一部分

**推荐指数**: ⭐⭐⭐

---

## 🎯 回归任务推荐

### 推荐方案：LightGBM

**理由**：

1. **专为回归优化**
   - 对连续值预测效果好
   - 损失函数针对回归设计
   - 能捕捉非线性关系

2. **性能优秀**
   - 训练速度快
   - 预测速度快
   - 准确率高

3. **适合股票预测**
   - 预测收益率（连续值）
   - 能处理220个因子
   - 支持大规模数据

4. **GPU加速**
   - 支持GPU训练
   - 大幅提升训练速度

## 💡 使用建议

### 1. 首选：LightGBM

```python

mode = 'regression'
n_estimators = 100
max_depth = 10
learning_rate = 0.1
num_leaves = 31
```

**适合**：
- 预测收益率
- 追求高准确率
- 有GPU资源
- 愿意花时间调参

```python
# 推荐配置

mode = 'regression'
```

**适合**：
- 不确定最佳模型

- 不想花时间调参

## 🚀 最佳实践：集成模型

### 推荐组合：LightGBM + RandomForest + Ridge

**优点**：
- 结合多个模型的优势
- LightGBM：高准确率
- RandomForest：稳定性
- Ridge：线性关系
- 提高预测稳定性
- 降低过拟合风险

**实现**：
```python
# 训练多个模型
models = {
    'LightGBM': train_model('LightGBM', 'regression'),
    'RandomForest': train_model('RandomForest', 'regression'),
    'Ridge': train_model('Ridge', 'regression')
}

# 预测时融合
predictions = []
for name, model in models.items():
    pred = model.predict(X)
    predictions.append(pred)

# 平均融合
final_prediction = np.mean(predictions, axis=0)
```
